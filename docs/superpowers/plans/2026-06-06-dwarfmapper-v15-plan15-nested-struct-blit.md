# DwarfMapper.NET — Plan 15: Nested-Struct Blit Recursion

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Relax the Plan 13 reinterpret-blit proof (currently primitives-only) so a struct field may itself be a **layout-identical nested struct**. This is the planned follow-up to the deliberate "primitives-only first" v1 decision — it widens the SIMD fast-path to real-world vector/matrix DTOs (`struct Transform { Vec3 Pos; Vec3 Scale; }`) while staying provably safe.

**Architecture:** Make `BlittableProof` recursive. A field pair is layout-identical when it is the same primitive `SpecialType`, the identical type, **or** two structs that recursively satisfy the same proof (unmanaged + source + `Sequential` + equal `Pack` + positional same-name + recursively-identical field types). C# forbids self-referential value types, so the recursion always terminates — no cycle guard needed. Soundness is preserved: a nested struct field is byte-copied positionally, which equals its name-based mapping precisely because the recursion proves names + layout align at every level.

**Tech Stack:** As Plans 1–14.

**Builds on Plan 13.** `BlittableProof.CanReinterpret(src,dst)` currently: identity→false; both `IsUnmanagedType` source structs; `Sequential`+equal `Pack`; ≥1 field; per-field **same name AND `IsPrimitive` of same `SpecialType`** (the primitives-only restriction this plan lifts). Helpers `IsPrimitive`, `InstanceFields`, `IsSourceSequential` stay. `CollectionConverter.SynthesizeBlit` + the auto-blit wiring + `[Reinterpret]` are unchanged. Diagnostics DWARF001–022. ~133 tests.

**Out of scope:** no new diagnostic, no attribute, no emitter change — purely a proof relaxation. List/span blit still out.

```csharp
    public static bool CanReinterpret(ITypeSymbol src, ITypeSymbol dst)
    {
        // Identity is the existing Clone() memmove, not a reinterpret.
        if (SymbolEqualityComparer.Default.Equals(src, dst))
        {
            return false;
        }
        return LayoutIdentical(src, dst);
    }

    /// <summary>
    /// True when two types are byte-identical in layout AND field-name-aligned, so a positional
    /// reinterpret equals DwarfMapper's name-based mapping. Recurses through nested structs.
    /// </summary>
    private static bool LayoutIdentical(ITypeSymbol a, ITypeSymbol b)
    {
        if (SymbolEqualityComparer.Default.Equals(a, b))
        {
            return true; // same type -> trivially same layout and name-aligned
        }
        if (!a.IsUnmanagedType || !b.IsUnmanagedType)
        {
            return false;
        }
        if (IsPrimitive(a) || IsPrimitive(b))
        {
            return a.SpecialType == b.SpecialType && a.SpecialType != SpecialType.None;
        }
        if (a is not INamedTypeSymbol na || b is not INamedTypeSymbol nb)
        {
            return false;
        }
        if (na.TypeKind != TypeKind.Struct || nb.TypeKind != TypeKind.Struct)
        {
            return false; // excludes enums (TypeKind.Enum): by-name enum mapping != byte copy
        }
        if (!IsSourceSequential(na, out var packA) || !IsSourceSequential(nb, out var packB))
        {
            return false;
        }
        if (packA != packB)
        {
            return false;
        }

        var fa = InstanceFields(na);
        var fb = InstanceFields(nb);
        if (fa.Count == 0 || fa.Count != fb.Count)
        {
            return false;
        }
        for (var i = 0; i < fa.Count; i++)
        {
            if (!string.Equals(fa[i].Name, fb[i].Name, System.StringComparison.Ordinal))
            {
                return false; // positional == name-based requires same names
            }
            if (!LayoutIdentical(fa[i].Type, fb[i].Type))
            {
                return false; // recurse: primitive same-SpecialType, identical type, or nested layout-identical struct
            }
        }
        return true;
    }
```

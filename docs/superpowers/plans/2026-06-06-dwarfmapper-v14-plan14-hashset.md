> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Map set members — source `HashSet<T>` / `ISet<T>` / `IReadOnlySet<T>` (anything `IEnumerable<T>`) → target `HashSet<U>` — converting elements through the full conversion machinery. Closes the obvious collection gap (we already do arrays, `List<T>`, `Dictionary<,>`).

**Architecture:** Extend `CollectionConverter` with a third `TargetKind.HashSet`, mirroring the `List` path. Detection adds `HashSet<U>` target; `Synthesize` adds a HashSet branch (identity → bulk `new HashSet<U>(src)`; else loop with `Add`). Element conversion already flows through the recursive `TryResolveConversion`. No new diagnostic.

**Tech Stack:** As Plans 1–13.

**Builds on Plan 6/13.** `CollectionConverter` has `internal enum TargetKind { Array, List }`, `Shape{Target,SourceIsArray,Count}`, `TryResolve`, `Synthesize`, `IsListT`, `Fq`, `Hash`, `ElementExpr`, `TryGetEnumerableElement`. The collection branch in `TryResolveConversion` calls `TryResolve` then (auto-blit for arrays) then recursive element resolve + `Synthesize`. Dictionary branch precedes it; blit needs `Target==Array`. Diagnostics DWARF001–022. ~130 tests.

**Out of scope:** `SortedSet`, `ImmutableHashSet`, target interface (`ISet` as destination), custom comparers.

    {
        element = t;
        if (t is INamedTypeSymbol n && n.TypeArguments.Length == 1
            && n.Name == "HashSet"
            && n.ContainingNamespace?.ToDisplayString() == "System.Collections.Generic")
